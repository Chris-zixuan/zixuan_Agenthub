# 支付宝交易流水记账 Skill 设计文档

## 概述

将支付宝导出的 CSV 交易明细，按照用户自定义的分类和账户映射规则，自动填充到随手记模板中，生成一份可直接导入记账网站的 Excel 文件。

## 文件清单

| 文件 | 角色 | 读写 |
|------|------|------|
| `template.xls` | 原始模板（记账网站认可的格式） | 只读，永不覆盖 |
| `template.xlsx` | Skill 工作文件，在原始模板基础上增加配置 sheet | 读写 |
| `支付宝交易明细(YYYYMMDD-YYYYMMDD).csv` | 支付宝导出的原始交易数据（GBK 编码） | 只读 |
| `输出-YYYYMMDD.xlsx` | Skill 生成的成品文件 | 写入 |

## template.xlsx 结构

共 5 个 sheet：

### 1. 支出 sheet（保留原模板）
列：交易类型, 日期, 一级分类, 二级分类, 支付账户, 金额, 成员, 商家, 项目, 备注

### 2. 收入 sheet（保留原模板）
列：交易类型, 日期, 一级分类, 二级分类, 收入账户, 金额, 成员, 商家, 项目, 备注

### 3. 转账 sheet（保留原模板）

## Skill 处理逻辑

### 输入
- `支付宝交易明细(YYYYMMDD-YYYYMMDD).csv`（取最新的一个）
- `template.xlsx`（含用户已填好的映射规则）

### 步骤

1. **初始化**
   - 读取 template.xlsx，加载映射规则
   - 加载 template.xls 作为基础结构参考（验证格式）

2. **解析 CSV**
   - 以 GBK 编码读取
   - 跳过前 24 行说明/表头，从第 25 行开始解析
   - 解析为结构化记录列表

3. **分类处理每条记录**

   ```
   对每条记录：
   收/支 = "支出"
     ├─ 状态含"退款"或交易分类="退款" → 金额取负，填支出 sheet
     ├─ 正常支出 → 填支出 sheet
   收/支 = "收入" → 填收入 sheet
   收/支 = "不计收支"
     ├─ 交易分类="投资理财"且金额≤0.04 → 忽略
     ├─ 状态含"关闭" → 忽略
     └─ 其他 → 根据用户映射处理
   ```

4. **填充到对应 sheet**
   - 根据分类映射确定 一级分类/二级分类
   - 根据账户映射确定 支付账户/收入账户
   - 清空模板中的示例数据行，写入实际数据

5. **输出**
   - 保存为 `输出-YYYYMMDD.xlsx`

### 边界情况处理
- 空记录 / 解析失败的行 → 跳过并记录日志
- 映射表中找不到匹配的分类 → 留空，用户后续补充
- 退款金额为负 → 支出 sheet 中显示负数
- 多个 CSV 文件 → 默认取最新一个

## 未纳入范围（明确不做）

- 不修改原始 template.xls
- 不做智能分类推断（分类权交给用户）
- 不生成图表或统计分析
- 不处理微信等其他支付平台的流水
