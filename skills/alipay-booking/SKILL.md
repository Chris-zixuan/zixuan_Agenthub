# 支付宝流水记账

一键处理支付宝 CSV → 随手记 Excel，Agent 自动推断分类映射。

## 用户触发方式

- "帮我整理账单"
- "处理支付宝流水"
- "/alipay-booking"

## 完整工作流

### 步骤 1：定位文件

1. 确认工作目录（用户指定，或当前目录）
2. 在工作目录中找 `支付宝交易明细*.csv`（取最新一个）
3. 如果找不到 CSV，询问用户提供路径

### 步骤 2：检查模板

检查工作目录中是否存在 `template.xlsx`：

- **已存在** → 继续步骤 3
- **不存在** → 运行初始化：
  ```bash
  python "<SKILL_DIR>/init_template.py" <工作目录>
  ```
  其中 `<SKILL_DIR>` 为本 skill 目录（`~/.claude/skills/alipay-booking/`）。
  生成的 `template.xlsx` 含预填的 CSV 类目和支付方式，一级/二级分类留空待 Agent 推断。

### 步骤 3：运行处理脚本

```bash
python "<SKILL_DIR>/process_alipay.py" <工作目录>
```

### 步骤 4：处理未映射类目（Agent 自动推断）

脚本输出中如果出现 `[!] 警告`，说明有 CSV 交易分类或支付方式在映射表中找不到。

**对未映射的交易分类：**
1. 根据脚本输出的示例交易（交易对方 + 商品说明）推断合理的 `(一级分类, 二级分类)`
2. 参考 `template.xlsx` 的「分类映射」sheet 中已有的分类法条目
3. 推断依据：类目名、交易对方、商品说明、已映射的相似类目
4. **直接给出推断建议让用户确认**，例如：

```
发现 2 个未映射的 CSV 交易分类，我已根据交易上下文推断：

- 「医疗教育」→ 一级: 医疗保健, 二级: 治疗费
  示例: 平安好医生 - 在线问诊
- 「美容美发」→ 一级: 医疗保健, 二级: 美容费
  示例: XX造型 - 剪发

是否有需要调整的？（没有的话我直接更新）
```

5. 用户确认后，用 openpyxl 更新 `template.xlsx` 的「分类映射」sheet（追加新行或填充空列）

**对未映射的支付方式：**
- 同样根据上下文推断对应账户
- 确认后更新「账户映射」sheet

6. 更新后重新运行步骤 3，确认警告消失

### 步骤 5：报告结果

输出文件 `输出-YYYYMMDD.xlsx` 生成后，告知用户：

- 支出 / 收入 / 转账各多少条
- 输出文件路径
- `template.xls` 未被修改（始终只读）

## 处理规则

- 退款：金额取负，填入支出 sheet（`is_refund` 自动检测"退款"关键词）
- 投资理财收益 ≤ 0.04 元：自动忽略
- 交易关闭：自动忽略（仅限"不计收支"记录）
- 组合支付（如 `账户余额&红包`）：自动取主账户（`&` 前部分）
- 合并支付的 `&` 分隔符会自动归一化

## 脚本位置

所有处理脚本位于本 skill 目录：

| 文件 | 用途 |
|------|------|
| `process_alipay.py` | 核心处理：CSV + 映射 → 输出 xlsx |
| `init_template.py` | 初始化：template.xls → template.xlsx + 映射 sheet |
| `template.xls` | 内置模板（fallback，用户工作目录的优先） |

## 文件保护

- `template.xls` — 记账网站专用格式，**只读不写，绝不覆盖**
- 输出文件命名 `输出-YYYYMMDD.xlsx`，每次独立生成
